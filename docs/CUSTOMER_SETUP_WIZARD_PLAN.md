# Customer Setup Wizard - Complete Implementation Plan

## Overview

**Goal:** Create a comprehensive wizard that guides users through complete customer onboarding, including customer data setup AND AI prompt generation in one seamless experience.

**Approach:** Step-by-step wizard with voice-first input, PDF context, and intelligent multi-account handling.

**Output:** Fully configured customer record with optimized AI prompts ready for order processing.

**Navigation:** Separate page accessed from main Settings page. **Existing customer form remains unchanged.**

---

## Important: Two Customer Creation Paths

### Path 1: Customer Setup Wizard (NEW)
- **Location:** Settings â†’ Customer Setup Wizard card â†’ `/settings/wizard`
- **Best for:** New customers, first-time setup, AI prompt generation
- **Features:** Guided 30+ steps, voice input, PDF upload, AI generation
- **Time:** 15-20 minutes

### Path 2: Quick Customer Form (EXISTING - UNCHANGED)
- **Location:** Settings â†’ Customers â†’ New Customer â†’ `/settings/customers/new`
- **Best for:** Advanced users, quick setup, manual prompt entry
- **Features:** Single form, all fields visible, manual configuration
- **Time:** 5 minutes (if you know what to do)

**See:** `WIZARD_NAVIGATION_STRUCTURE.md` for complete navigation details and UI designs.

### New Table: `customer_child_accounts`

```sql
CREATE TABLE customer_child_accounts (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  customer_ps_id VARCHAR(50) NOT NULL, -- Parent customer (should be "MULTI")
  child_ps_account_id VARCHAR(50) NOT NULL,
  routing_description TEXT NOT NULL, -- User's description of when to use this account
  display_order INTEGER NOT NULL DEFAULT 1,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),

  CONSTRAINT fk_customer FOREIGN KEY (customer_ps_id)
    REFERENCES customers(ps_customer_id) ON DELETE CASCADE,
  UNIQUE(customer_ps_id, child_ps_account_id)
);

CREATE INDEX idx_child_accounts_customer ON customer_child_accounts(customer_ps_id);

COMMENT ON TABLE customer_child_accounts IS
  'Stores child PeopleSoft account IDs for multi-account customers.
   Parent customer has ps_customer_id = "MULTI".';
```
